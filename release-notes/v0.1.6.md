## Changes

- Added official withdrawn-release policy support so SEAJelly can mark a bad release as removed from the upgrade path without rewriting git history
- Added bridge-release support in the updater so future releases can reason about safe predecessor tags more explicitly
- Published `v0.1.6` as the bridge fix for installs still on `v0.1.3`, carrying forward the `v0.1.4` database change plus the `v0.1.5` Updates-page repair

## Notes

- `v0.1.4` is now considered withdrawn from the official upgrade path
- Fresh `v0.1.3` installs should upgrade directly to `v0.1.6`
- Installs already stuck on `v0.1.4` waiting for a database update still need one manual recovery action to finish that paused run
