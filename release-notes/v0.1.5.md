## Changes

- Fixed the Updates page so releases waiting on a manual database step now show an explicit `Apply Database Update` action in the current-run card
- Stopped unnecessary polling while a run is paused at `db_pending`, which avoids hiding the follow-up database action behind a misleading “upgrade still running” state

## Notes

- Existing installs that were stuck on `Database update pending` do not need a reinstall
- After updating to this release, those installs can finish the paused upgrade directly from `Dashboard -> Updates`
